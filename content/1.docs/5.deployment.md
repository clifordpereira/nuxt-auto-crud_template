> The `nuxt-auto-crud stack` is optimized for **Cloudflare D1** and **NuxtHub** deployment. Its **Zero-Codegen** architecture ensures that production environments stay synchronized with the Drizzle schema via remote migration applications.

## üõ† Deployment Workflow [Template Implementation]

### 1. Cloudflare D1 Provisioning
Create the production database instance in the Cloudflare dashboard.
```bash
npx wrangler d1 create <project-db-name>
```

### 2. Configuration (`wrangler.toml`)
Bind the production database ID to your Nuxt application.
```toml
[[d1_databases]]
binding = "DB"
database_name = "nuxt-crud-prod"
database_id = "xxxx-xxxx-xxxx-xxxx"
```

### 3. Remote Migrations
Apply your Drizzle schema to the remote D1 instance.
```bash
npx wrangler d1 migrations apply <project-db-name> --remote
```

### 4. Admin Initialization [Template Implementation]
Deploy the application and visit `/login`. The first login with the `NUXT_ADMIN_EMAIL` will trigger the `db:seed` task to hydrate roles and permissions.

## üîê Production Environment Variables
Set these via the Cloudflare dashboard or `wrangler secret`.

| Variable | Importance | Description |
| :--- | :--- | :--- |
| `NUXT_SESSION_PASSWORD` | **Critical** | Minimum 32 characters for session encryption. |
| `NUXT_ADMIN_EMAIL` | **Required** | Seed target for the root admin user. |
| `NUXT_ADMIN_PASSWORD` | **Required** | Initial credential for first-access hydration. |

## üö¶ Post-Deployment Checklist
1.  **Schema Check**: Verify migrations applied correctly via `wrangler d1 query`.
2.  **Auth Check**: Confirm `NUXT_SESSION_PASSWORD` is unique to production.
3.  **Permissions**: Use the template's Admin UI to finalize production-specific RBAC rules.
